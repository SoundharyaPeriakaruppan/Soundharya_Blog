Git cheatsheet for testers

Git is a popular version control system that allows developers to track changes in their code and collaborate with others. If you are new to Git, it can be overwhelming to remember all the commands and options available. This cheatsheet provides a quick reference guide for some of the most commonly used Git commands.

Clone a Repository

git clone (repository_url)

Check the Status of Your Repository

git status

Add Changes to Stage

git add .

or add specific file:

git add (file_name)

Commit Changes

git commit -m "commit message"

Push Changes to Remote Repository

git push origin (branch_name)

Pull Latest Changes from Remote Repository

git pull origin (branch_name)

Create a New Branch

git branch (branch_name)

Switch to a Different Branch

git checkout (branch_name)

Merge Branches

git merge (source_branch)

Delete a Branch

git branch -d (branch_name)

View Commit History

git log

Revert to a Previous Commit

git revert (commit_id)

Remember to replace (repository_url), (file_name), (commit message), (branch_name), and (commit_id) with your actual values.

Also, ensure you are in the correct directory in your terminal or command prompt that contains the Git repository before running these commands.

Note: This is a basic cheatsheet. Git has many more commands for different purposes.
